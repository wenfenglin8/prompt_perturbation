# RQ2 150-Item Expanded Analysis

This report reruns the RQ2 drift-performance analysis after combining the original n50 GPT/main data with the add100 GPT/main data. The design contains 150 items per task for factual QA, math reasoning, and code generation; each item has five original generations and five generations under each of five perturbation types.

Final item-perturbation table: n = 2250 observations (3 tasks x 150 items x 5 perturbation types). Semantic drift is measured as noise-corrected Sentence-BERT cosine distance. Performance change is original performance minus perturbed performance, so positive values indicate a performance decrease.

## Overall Drift-Performance Association

Pooling all 2250 observations, Spearman rho = 0.105 (p = <0.001; Holm-adjusted p = <0.001). Pearson r = 0.167 (p = <0.001). As in the smaller RQ2 analysis, Pearson and Spearman diverge, indicating that the association is sensitive to distribution shape and high-leverage cells rather than a stable monotone relationship across all perturbations and tasks.

## Factual QA Robustness Check

For factual QA, the raw pooled Spearman association across all perturbations is rho = 0.234 (p = <0.001). This is the closest direct analogue of the earlier n=50 factual QA result, but now uses 150 factual QA items x 5 perturbations = 750 observations.

After adding perturbation fixed effects, the drift coefficient is beta = 0.674 with naive OLS p = <0.001. With item-clustered standard errors and a t(G-1) reference distribution over G = 150 item clusters, beta = 0.674, SE = 0.291, t = 2.314, p = 0.022. The asymptotic normal cluster p-value is 0.021. Unlike the earlier 10-cluster robustness check, the expanded sample has enough clusters for the cluster-robust approximation to be less fragile.

Within-perturbation centering gives Spearman rho = 0.213 (naive p = <0.001), describing the residual continuous association after subtracting perturbation-type means.

## Answer to RQ2

The expanded 150-item-per-task analysis gives a more stable estimate than the earlier 10-item-per-task RQ2 run. The pooled all-task association remains weak under Spearman, so RQ2 should not be answered as a simple global monotone relationship between semantic drift and performance decrease. The strongest evidence remains task-conditional: factual QA shows a positive drift-performance association, and the larger item count makes the perturbation-controlled, item-clustered robustness check more credible than in the original small-sample analysis. Math reasoning and code generation should be interpreted from their task-specific rows rather than inferred from the pooled trend.

Paraphrasing remains the main perturbation to inspect because it tends to create larger semantic drift and larger mean performance changes than the other perturbation types, but the report keeps the multiplicity-adjusted p-values visible. The cleanest conclusion is therefore conditional rather than universal: semantic drift is most clearly informative for factual QA, while cross-task pooling masks heterogeneous behavior across tasks and perturbation families.
